# Telemetry Statistics Script

The EDR Telemetry statistics data is generated by:

```text
scripts/generate_telemetry_stats.py
```

The script reads the Git history of this repository and compares the telemetry matrix files before and after each relevant commit. It records one event for each changed telemetry cell.

A telemetry cell is:

```text
platform + vendor + telemetry category + telemetry sub-category + status
```

For example, if a vendor changes from `No` to `Yes` for a Windows telemetry field, the script records that exact change and links it back to the commit or pull request that introduced it.

## What The Script Measures

The script measures changes in telemetry visibility exposed to users. It does not measure product quality, prevention capability, detection quality, alert quality, managed service quality, or incident-response capability.

## How Scores Are Calculated

The script uses the same status scoring model as the project:

| Status | Score |
| --- | ---: |
| `Yes` | 1.0 |
| `Via EnablingTelemetry` | 1.0 |
| `Partially` | 0.5 |
| `Via EventLogs` | 0.5 |
| `No` | 0.0 |
| `Pending Response` | 0.0 |

Each changed cell is weighted by the telemetry feature weight from `Tools/compare.py`.

```text
Weighted change = (new status score - old status score) x feature weight
```

This means a change from `No` to `Yes` increases the score, while a change from `Yes` to `No` decreases it. Changes such as `Via EnablingTelemetry` to `Yes` are also tracked because they improve default telemetry availability, even when the official numeric score stays the same.

## What The Script Does Not Count As Improvement

The script avoids inflating improvement statistics by labeling changes that are not ordinary coverage gains, including:

- new vendor baselines
- new telemetry categories
- vendor renames
- documentation-only changes
- tooling-only changes
- corrections or downgrades
- open pull requests that have not been merged

These changes are still recorded for transparency, but they are not treated as normal vendor improvements.

## Generated Files

The main output is:

```text
data/generated/telemetry_change_events.jsonl
```

This is the detailed ledger with one row per changed telemetry cell.

The script also creates summary files for the website dashboards, including:

```text
data/generated/current_vendor_scores.json
data/generated/vendor_score_timeseries.json
data/generated/vendor_change_summary.json
data/generated/category_change_summary.json
data/generated/contributor_summary.json
data/generated/pr_summary.json
data/generated/direct_commit_summary.json
data/generated/manual_review_items.json
data/generated/run_metadata.json
```

## Running The Script

Run from a full Git checkout:

```bash
python scripts/generate_telemetry_stats.py \
  --repo . \
  --output data/generated \
  --raw-output data/raw/github \
  --config config/stats_config.yml \
  --format json,jsonl,csv
```

If `GITHUB_TOKEN` is available, the script can enrich results with pull request metadata. Without it, the script still produces deterministic statistics from local Git history.
